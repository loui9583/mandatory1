Git in the Terminal

Introduction to Basic Git Commands in the Terminal

Git is a widely used version control system that allows developers to track changes in their projects and collaborate with others efficiently. While there are several graphical user interfaces (GUIs) available for Git, learning how to use Git in the terminal provides a deeper understanding of Git's functionality and enhances your command-line skills.

Initializing a Repository

To start using Git in a project, navigate to the project's directory in your terminal and initialize a Git repository with the following command:

$ git init

Adding and Committing Changes

After making changes to your project files, you can add them to the staging area using:

$ git add filename

Once files are staged, commit them to the repository:

$ git commit -m "Commit message"

If you want to commit all changes, including deletions, you can use the following command, which stages all tracked changes (modified and deleted) before creating a new commit:

$ git commit -a "Commit message"

**Important Note:** While `git commit -a` is convenient, it's essential to be aware that it stages changes automatically. If you have unstaged changes you don't want to commit, using `git commit -a` might capture them unintentionally. It's a good practice to review the changes using `git status` before committing to ensure you're capturing what you intend to.

Checking the Status and History

You can check the current status of your repository using:

$ git status

To view the commit history:

$ git log

Branching and Merging

Branching allows you to create isolated workspaces within your Git repository. This is useful for developing new features or fixing bugs without affecting the main codebase. Here are some basic branching commands:

$ git branch branchname (to create a new branch)
$ git checkout branchname (to switch to a different branch)
$ git merge branchname (to merge changes from one branch into another)

Ignoring Files with .gitignore

The `.gitignore` file specifies intentionally untracked files that Git should ignore. These can include files generated by your operating system, code editor, or build tools, as well as sensitive information like API keys.

To create a `.gitignore` file in your project, simply create a new file named `.gitignore` in the root of your project directory.

For example, in a Node.js project, your `.gitignore` file might look like this:


            /node_modules 

            *.log    

            .env

These patterns specify that files and directories matching these patterns should be ignored by Git. This helps keep your repository clean and focused on the files that are important for version control.

Pushing Changes

Once you've made and committed changes to a local branch, you can push them to a remote repository (like GitHub) to share your work with others or collaborate on a project. To push changes to a remote repository:

$ git push origin branchname

These are just a few basic commands to get started with Git in the terminal. As you become more familiar with Git, you'll discover more advanced features and techniques to streamline your workflow.

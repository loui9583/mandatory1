Terminology for different parts of an URL

Understanding the various components of a URL

A URL (Uniform Resource Locator) is a reference to a web resource that specifies its location on a computer network and a mechanism for retrieving it. It consists of several parts:

  1. Scheme: The scheme indicates the protocol used to access the resource. Common schemes include http, https, ftp, etc.
  2. Authority: The authority component typically consists of two subcomponents:
    • Host: The host name or IP address of the server hosting the resource.
    • Port: An optional port number where the resource is hosted. If omitted, the default port for the scheme is used (e.g., 80 for HTTP, 443 for HTTPS).
  3. Path: The path component specifies the specific resource on the server. It may include directories and file names.
  4. Query: The query component contains data to be passed to the server as part of the URL. It typically consists of key-value pairs separated by ampersands (&).

Here's an example URL broken down into its components:


                URL: https://www.example.com:8080/path/to/resource?key1=value1&key2=value2#section1
                Scheme: https
                Host: www.example.com
                Port: 8080
                Path: /path/to/resource
                Query: key1=value1&key2=value2
